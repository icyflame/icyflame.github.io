Programming: Rails, Node.js, ExpressJS, HTML, HAML, REST APIs …

I like Ruby and Rails. Rails is not too big to be a drag when building small
applications, and not small enough (like ExpressJS) to have to write everything.
Implicit logic is better than having to write everything and configure
everything (like Django). I am a huge fan of the Convention over Configuration
approach to software. I have built a few apps in Rails, and I would like to talk
about one in particular. It is a pet project of mine, that didn’t gain as much
traction as I hoped it would, but I am proud of it, nonetheless:
http://cutouts.herokuapp.com.

Cutouts is a simple no-bells-and-whistles link aggregator. It doesn’t
concentrate on Images (Pinterest), it was a lot like Kippt till it shutdown, It
doesn’t have any extension at all. You go to the website, paste the link, write
the author’s name and paste a quote from the article that you think is
absolutely profound or insightful or thoughtful or quote-worthy and then, you
save it. That’s it. You can export an HTML webpage of all the articles you
saved if you ever want to share all the things you read and loved on the
internet. It’s built with rails and is completely open source on GitHub:
https://github.com/icyflame/cutouts I had a few people contribute to this
project and that was a really fulfilling experience, to see someone spend enough
time to understand and contribute to my project! I have used a few gems in
this project: Devise for Auth, Bootstrap for the UI, Rails Admin for the admin
dashboard, Font awesome for the icons.

Another huge Rails application that I have built is ERP Rails which is a
custom-built ERP system for managing networking, sponsorship calling inside
Students’ Alumni Cell: https://github.com/icyflame/erp-rails At one point in
time, the data of around 2000 people was there in the database. They were reached
out to and convinced to attend the Alumni meet using this application. It has
provisions to store contact details, summary for calls and dates for the
next call and a dashboard that lets people in a supervising role look
into whether or not their subordinates are achieving the goals set for them.
